<p align="justify"><b>Sistem Operasi Disk</b></br>Komputer pertama tidak memiliki perangkat penyimpanan modern seperti hard drive, drive optik, atau penyimpanan flash. Floppy disk dan penyimpanan hard disk memerlukan perangkat lunak untuk membaca, menulis, dan mengelola data yang disimpannya. Disk Operating System (DOS) adalah sistem operasi yang digunakan komputer untuk mengaktifkan perangkat penyimpanan data ini untuk membaca dan menulis file. DOS menyediakan sistem file yang mengatur file dengan cara tertentu pada disk. Microsoft membeli DOS dan mengembangkan MS-DOS. MS-DOS menggunakan baris perintah sebagai antarmuka bagi orang-orang untuk membuat program dan memanipulasi file data, seperti yang ditunjukkan pada keluaran perintah. Perintah DOS ditampilkan dalam teks tebal.</p>

 <p align="justify"><b>Versi Windows</b></br>
Sejak 1993, ada lebih dari 20 rilis Windows yang didasarkan pada sistem operasi NT. Sebagian besar versi ini untuk digunakan oleh masyarakat umum dan bisnis karena keamanan file yang ditawarkan oleh sistem file yang digunakan oleh OS NT. Bisnis juga mengadopsi sistem operasi Windows berbasis OS NT. Ini karena banyak edisi dibuat khusus untuk workstation, profesional, server, server tingkat lanjut, dan server pusat data, untuk menyebutkan beberapa dari banyak versi yang dibuat khusus.</br></br>
Dimulai dengan Windows XP, edisi 64-bit telah tersedia. Sistem operasi 64-bit adalah arsitektur yang sama sekali baru. Itu memiliki ruang alamat 64-bit, bukan ruang alamat 32-bit. Ini bukan hanya dua kali jumlah ruang karena bit ini adalah bilangan biner. Sementara Windows 32-bit dapat menangani RAM kurang dari 4 GB, Windows 64-bit secara teoritis dapat menangani 16,8 juta terabyte. Ketika OS dan perangkat keras semuanya mendukung operasi 64-bit, kumpulan data yang sangat besar dapat digunakan. Kumpulan data yang besar ini mencakup database yang sangat besar, komputasi ilmiah, dan manipulasi video digital definisi tinggi dengan efek khusus. Secara umum, komputer dan sistem operasi 64-bit kompatibel dengan versi sebelumnya, program 32-bit, tetapi program 64-bit tidak dapat dijalankan pada perangkat keras 32-bit yang lebih lama.</br></br>
Dengan setiap rilis Windows berikutnya, sistem operasi menjadi lebih disempurnakan dengan memasukkan lebih banyak fitur. Windows 7 ditawarkan dengan enam edisi berbeda, Windows 8 sebanyak lima, dan Windows 10 dengan delapan edisi berbeda! Setiap edisi tidak hanya menawarkan kemampuan yang berbeda, tetapi juga harga yang berbeda. Microsoft telah mengatakan bahwa Windows 10 adalah versi terakhir Windows, dan Windows telah menjadi layanan, bukan hanya OS. Mereka mengatakan bahwa daripada membeli sistem operasi baru, pengguna hanya akan memperbarui Windows 10 sebagai gantinya.</p>

<p align="justify"><b>Lapisan Abstraksi Perangkat Keras</b></br>
Komputer Windows menggunakan berbagai jenis perangkat keras. Sistem operasi dapat diinstal di komputer yang dibeli atau di komputer yang dirakit oleh pengguna. Ketika sistem operasi diinstal, itu harus diisolasi dari perbedaan perangkat keras. Arsitektur dasar Windows ditunjukkan pada gambar.</p>

<p align="center">
  <img src="https://github.com/yenysyafitry/Cisco-CCNA/blob/main/Screenshot_7.jpg"> </p>
 
<p align="justify">Lapisan abstraksi perangkat keras (HAL) adalah perangkat lunak yang menangani semua komunikasi antara perangkat keras dan kernel. Kernel adalah inti dari sistem operasi dan memiliki kendali atas seluruh komputer. Ini menangani semua permintaan input dan output, memori, dan semua periferal yang terhubung ke komputer.</br>
Dalam beberapa kasus, kernel masih berkomunikasi dengan perangkat keras secara langsung, sehingga tidak sepenuhnya independen dari HAL. HAL juga membutuhkan kernel untuk menjalankan beberapa fungsi.</br></br>
<b>Mode Pengguna dan Mode Kernel</b></br>
Seperti yang ditunjukkan pada gambar, ada dua mode berbeda di mana CPU beroperasi ketika komputer telah menginstal Windows: mode pengguna dan mode kernel.</p>
<p align="center"><img src="https://github.com/yenysyafitry/CCNA-Certification-Training-Videos/blob/main/Screenshot_8.jpg"> </p>
 <p align="justify">
 Aplikasi yang diinstal berjalan dalam mode pengguna, dan kode sistem operasi berjalan dalam mode kernel. Kode yang dijalankan dalam mode kernel memiliki akses tidak terbatas ke perangkat keras yang mendasarinya dan mampu menjalankan instruksi CPU apa pun. Kode mode kernel juga dapat merujuk ke alamat memori apa pun secara langsung. Umumnya disediakan untuk fungsi OS yang paling tepercaya, crash dalam kode yang berjalan dalam mode kernel menghentikan operasi seluruh komputer. Sebaliknya, program seperti aplikasi pengguna, berjalan dalam mode pengguna dan tidak memiliki akses langsung ke perangkat keras atau lokasi memori. Kode mode pengguna harus melalui sistem operasi untuk mengakses sumber daya perangkat keras. Karena isolasi yang disediakan oleh mode pengguna, kerusakan dalam mode pengguna dibatasi hanya untuk aplikasi dan dapat dipulihkan. Sebagian besar program di Windows berjalan dalam mode pengguna.</br></br> 
Semua kode yang berjalan dalam mode kernel menggunakan ruang alamat yang sama. Driver mode kernel tidak memiliki isolasi dari sistem operasi. Jika terjadi kesalahan dengan driver yang berjalan dalam mode kernel, dan itu menulis ke ruang alamat yang salah, sistem operasi atau driver mode kernel lainnya dapat terpengaruh secara negatif. Dalam hal ini, pengemudi mungkin macet, menyebabkan seluruh sistem operasi macet.</br></br>
Ketika kode mode pengguna berjalan, itu diberikan ruang alamat terbatasnya sendiri oleh kernel, bersama dengan proses yang dibuat khusus untuk aplikasi. Alasan fungsionalitas ini terutama untuk mencegah aplikasi mengubah kode sistem operasi yang berjalan pada waktu yang sama. Dengan memiliki prosesnya sendiri, aplikasi tersebut memiliki ruang alamat pribadinya sendiri, sehingga aplikasi lain tidak dapat mengubah data di dalamnya. Ini juga membantu mencegah sistem operasi dan aplikasi lain agar tidak mogok jika aplikasi tersebut macet.</br></br>

 <p align="justify">NTFS adalah sistem file yang paling banyak digunakan untuk Windows karena berbagai alasan. NTFS mendukung file dan partisi yang sangat besar dan sangat kompatibel dengan sistem operasi lain. NTFS juga sangat andal dan mendukung fitur pemulihan. Yang terpenting, ini mendukung banyak fitur keamanan. Kontrol akses data dicapai melalui deskriptor keamanan. Penjelasan keamanan ini berisi kepemilikan file dan izin sampai ke tingkat file. NTFS juga melacak banyak stempel waktu untuk melacak aktivitas file. Sebelum perangkat penyimpanan seperti disk dapat digunakan, perangkat tersebut harus diformat dengan sistem file. Setiap partisi adalah unit penyimpanan logis yang dapat diformat untuk menyimpan informasi, seperti file data atau aplikasi. Selama proses instalasi, kebanyakan sistem operasi secara otomatis mempartisi dan memformat ruang drive yang tersedia dengan sistem file seperti NTFS.</br></br>
Pemformatan NTFS membuat struktur penting pada disk untuk penyimpanan file, dan tabel untuk merekam lokasi file:</p>
<ul align="justify">	<li>

<li>File Sistem - Ini adalah file tersembunyi yang menyimpan informasi tentang volume lain dan atribut file.</li>

Penghapusan aman akan menulis data ke seluruh drive beberapa kali untuk memastikan tidak ada data yang tersisa.</p>

<p align="justify">Bab ini membahas beberapa jenis media jaringan, termasuk kabel twisted-pair, kabel koaksial, kabel serat optik, dan nirkabel. <b>Kabel dan Infrastruktur</b></br>
Media adalah lingkungan fisik aktual tempat data bergerak saat bergerak dari satu komponen ke komponen lainnya, dan menghubungkan perangkat jaringan. Jenis media jaringan yang paling umum adalah kabel twisted-pair, kabel koaksial, kabel serat optik, dan nirkabel. Setiap jenis media memiliki kemampuan khusus dan melayani tujuan tertentu.</br></br>
Memahami jenis koneksi yang dapat digunakan dalam jaringan memberikan pemahaman yang lebih baik tentang bagaimana jaringan berfungsi dalam mentransmisikan data dari satu titik ke titik lainnya.</br></br>
<b>Kabel Twisted-Pair</b></br>
Twisted-pair adalah kabel berbasis kawat tembaga yang dapat diberi pelindung atau tidak. Twisted-pair adalah media yang paling umum untuk konektivitas jaringan.</br></br>
<b>Kabel unshielded twisted-pair (UTP)</b>, seperti yang ditunjukkan pada Gambar 4-1, adalah kabel empat pasang. Masing-masing dari delapan kabel tembaga individu di kabel UTP ditutupi oleh bahan isolasi. Selain itu, kabel di setiap pasangan dipelintir satu sama lain. Keunggulan kabel UTP adalah kemampuannya untuk menghilangkan interferensi, karena pasangan kabel-twisted membatasi degradasi sinyal dari interferensi elektromagnetik (EMI) dan interferensi frekuensi radio (RFI). Untuk lebih mengurangi crosstalk antara pasangan kabel UTP, jumlah lilitan pada pasangan kabel bervariasi. UTP, serta kabel berpelintir berpelindung (STP), harus mengikuti spesifikasi yang tepat mengenai berapa banyak lilitan atau kepang yang diizinkan per meter.</p>

<p align="justify"><b>Kabel Twisted-Pair Tanpa Pelindung</b></br>
Kabel UTP digunakan di berbagai jaringan. Saat digunakan sebagai media jaringan, kabel UTP memiliki empat pasang kabel tembaga ukuran 22 atau 24. UTP yang digunakan sebagai media jaringan memiliki impedansi 100 ohm, yang membedakannya dari jenis kabel twisted-pair lainnya seperti yang digunakan untuk kabel telepon. Karena kabel UTP memiliki diameter luar kira-kira 0,43 cm (0,17 inci), ukurannya yang kecil dapat menguntungkan selama pemasangan. Selain itu, karena UTP dapat digunakan dengan sebagian besar arsitektur jaringan utama, popularitasnya terus meningkat.</br></br>
Ada beberapa kategori kabel UTP:</p><ul align="justify"><li> 
Kategori 1 —Digunakan untuk komunikasi telepon; tidak cocok untuk transmisi data</li>
<li> Kategori 2 —Mampu mentransmisikan data dengan kecepatan hingga 4 Mbps</li>
<li> Kategori 3 —Digunakan dalam jaringan 10BASE-T; dapat mengirimkan data dengan kecepatan hingga 10 Mbps</li>
<li> Kategori 4 —Digunakan dalam jaringan Token Ring; dapat mengirimkan data dengan kecepatan hingga 16 Mbps</li>
<li> Kategori 5 —Mampu mentransmisikan data dengan kecepatan hingga 100 Mbps</li>
<li> Kategori 5e —Digunakan dalam jaringan yang berjalan dengan kecepatan hingga 1000 Mbps (1 Gbps)</li>
<li> Kategori 6 —Terdiri dari empat pasang kabel tembaga ukuran 24 yang dapat mengirimkan data dengan kecepatan hingga 1000 Mbps</li></ul>
<p align="justify"><b>Kabel Shielded Twisted-Pair (STP)</b>, menggabungkan teknik pelindung dan pelintiran kabel untuk lebih melindungi dari degradasi sinyal. Setiap pasang kabel dibungkus dengan lembaran logam. Keempat pasang kabel tersebut kemudian dibungkus dengan jalinan logam atau foil secara keseluruhan, biasanya kabel 150-ohm. Ditentukan untuk digunakan dalam penginstalan jaringan Ethernet, STP mengurangi gangguan listrik baik di dalam kabel (sambungan pasangan-ke-pasangan, atau crosstalk) dan dari luar kabel (EMI dan RFI). Topologi jaringan Token Ring menggunakan STP.</p>

<p align="justify"><b>Kabel Twisted-Pair Terlindung</b></br>
Saat Anda mempertimbangkan untuk menggunakan UTP dan STP untuk media jaringan Anda, pertimbangkan hal berikut:</p>
<ul align="justify"><li> Kecepatan dari kedua jenis media biasanya memuaskan untuk jarak area lokal.</li>
<li>Keduanya adalah media komunikasi data yang paling murah. UTP lebih murah dari STP.</li>
<li>Karena sebagian besar bangunan telah terhubung dengan UTP, banyak standar transmisi yang diadaptasi untuk digunakan guna menghindari pengkabelan ulang yang mahal dengan jenis kabel alternatif.</li></ul>

<p align="justify"><b>Kabel twisted-pair</b> adalah kabel jaringan yang paling umum digunakan saat ini; namun, beberapa jaringan masih menggunakan teknologi lama seperti kabel koaksial, seperti yang dibahas di bagian selanjutnya.</br></br>
<b>Kawat koaksial</b></br>
Kabel koaksial terdiri dari konduktor silinder luar berongga yang mengelilingi satu elemen penghantar kawat dalam. Bagian ini menjelaskan karakteristik dan penggunaan kabel koaksial.</P>
<p align="center"><img src="https://github.com/yenysyafitry/CCNA-Certification-Training-Videos/blob/main/fig02.gif"> </p>	 
<p align="justify">Kabel dalam tunggal yang terletak di tengah kabel koaksial adalah konduktor tembaga, dikelilingi oleh lapisan insulasi fleksibel. Di atas bahan isolasi ini terdapat anyaman anyaman tembaga atau lembaran logam yang berfungsi sebagai kabel kedua di sirkuit dan sebagai pelindung untuk konduktor bagian dalam. Lapisan atau perisai kedua ini dapat membantu mengurangi jumlah gangguan dari luar. Jaket luar menutupi perisai ini. Konektor BNC yang ditampilkan sangat mirip dengan konektor televisi kabel dan terhubung ke NIC lama dengan antarmuka BNC.</p>
<p align="justify"><b>Kabel Koaksial</b></br>
Kabel koaksial mendukung 10 hingga 100 Mbps dan relatif murah, meskipun lebih mahal daripada UTP. Kabel koaksial dapat diletakkan pada jarak yang lebih jauh dari kabel twisted-pair. Misalnya Ethernet dapat berjalan kira-kira 100 meter menggunakan kabel twisted pair, tetapi 500 meter menggunakan kabel coaxial.</br></br>
Kabel koaksial menawarkan beberapa keuntungan untuk digunakan di LAN. Ini dapat dijalankan dengan lebih sedikit dorongan dari repeater, yang meregenerasi sinyal dalam jaringan sehingga mereka dapat menjangkau jarak yang lebih jauh antara node jaringan daripada kabel STP atau UTP. Kabel koaksial lebih murah daripada kabel serat optik, dan teknologinya terkenal. Ini telah digunakan selama bertahun-tahun untuk semua jenis komunikasi data.</br></br>
Saat Anda bekerja dengan kabel, pertimbangkan ukurannya. Seiring dengan bertambahnya ketebalan atau diameter kabel, begitu pula kesulitan dalam mengerjakannya. Kabel harus sering ditarik melalui saluran dan palung yang ada yang ukurannya terbatas. Kabel koaksial tersedia dalam berbagai ukuran. Diameter terbesar, sering disebut sebagai Thicknet, ditentukan untuk digunakan sebagai kabel tulang punggung Ethernet karena secara historis memiliki panjang transmisi yang lebih besar dan karakteristik penolakan derau. Namun, kabel Thicknet mungkin terlalu kaku untuk dipasang dengan mudah di beberapa lingkungan karena ketebalannya. Umumnya, semakin sulit media jaringan dipasang, semakin mahal biaya pemasangannya. Kabel koaksial lebih mahal untuk dipasang daripada kabel twisted-pair, dan kabel Thicknet hampir tidak pernah digunakan kecuali untuk instalasi tujuan khusus, di mana pelindung dari EMI atau jarak memerlukan penggunaan kabel semacam itu.</br></br>
Di masa lalu, kabel koaksial dengan diameter luar hanya 0,35 cm, kadang-kadang disebut sebagai Thinnet , digunakan dalam jaringan Ethernet. Ini sangat berguna untuk instalasi kabel yang membutuhkan kabel untuk membuat banyak tikungan dan belokan. Karena Thinnet lebih mudah dipasang, itu juga lebih murah untuk dipasang. Jadi, itu juga disebut sebagai Cheapernet. Namun, karena jalinan tembaga atau logam bagian luar dalam kabel koaksial terdiri dari setengah sirkuit listrik, perhatian khusus perlu diberikan untuk mengardekannya dengan benar, dengan memastikan bahwa sambungan listrik yang kuat ada di kedua ujung kabel. Pemasang sering kali gagal membuat koneksi yang baik. Masalah koneksi mengakibatkan gangguan listrik, yang mengganggu transmisi sinyal. Karena alasan ini, meskipun diameternya kecil, Thinnet tidak lagi umum digunakan di jaringan Ethernet.</br></br>Meskipun kabel koaksial menawarkan beberapa keunggulan jarak dibandingkan twisted-pair, kerugiannya jauh lebih besar daripada manfaatnya. Jika sinyal komunikasi perlu menempuh jarak yang lebih jauh dengan kecepatan tinggi, lebih umum menggunakan kabel serat optik.</p>
<p align="justify"><b>Kabel serat optik</b></br>
Kabel serat optik adalah media jaringan yang mampu melakukan transmisi cahaya termodulasi. Bagian ini menjelaskan jenis, karakteristik, dan penggunaan kabel serat optik. Kabel serat optik yang digunakan untuk jaringan terdiri dari dua serat yang terbungkus dalam selubung terpisah. Melihat penampang melintang pada Gambar 4-4, Anda dapat melihat bahwa setiap serat optik dikelilingi oleh lapisan bahan penyangga pelindung: biasanya pelindung plastik, kemudian plastik seperti Kevlar, dan terakhir, jaket luar yang memberikan perlindungan bagi seluruh kabel. Plastik tersebut sesuai dengan kode kebakaran dan bangunan yang sesuai. Tujuan dari Kevlar adalah untuk memberikan bantalan dan perlindungan tambahan untuk serat kaca yang rapuh dan setipis rambut. Jika kabel serat optik yang terkubur diwajibkan oleh kode, kabel baja tahan karat terkadang disertakan untuk menambah kekuatan. Beberapa konektor dapat menghubungkan serat ke perangkat jaringan; yang paling umum adalah konektor SC, yang memiliki dua optik, satu menghubungkan untuk mengirim dan yang lainnya menghubungkan untuk menerima.</br> <img src="https://github.com/yenysyafitry/CCNA-Certification-Training-Videos/blob/main/fig04.gif"></br>
<b>Kabel Serat Optik</b></br>
Bagian pemandu cahaya dari serat optik disebut inti dan kelongsong . Inti kaca biasanya sangat murni dengan indeks bias yang tinggi. Ketika lapisan kelongsong kaca atau plastik dengan indeks bias rendah mengelilingi inti kaca, cahaya dapat terperangkap di inti serat. Proses ini disebut refleksi internal total , dan memungkinkan serat optik bertindak seperti pipa ringan, memandu cahaya untuk jarak yang jauh, bahkan di sekitar tikungan. Kabel serat optik adalah yang paling mahal dari tiga jenis yang dibahas dalam pelajaran ini, tetapi kabel ini mendukung kecepatan jalur tarif yang lebih tinggi.</br></br>
Kabel serat optik tidak membawa impuls listrik seperti kabel tembaga. Sebaliknya, sinyal yang mewakili bit diubah menjadi pulsa cahaya. Ada dua jenis kabel serat optik:</p>
<ul align="justify"><li>Single-mode</br> Single-mode fiber-optic cable hanya memungkinkan satu mode (atau panjang gelombang) cahaya untuk merambat melalui fiber. Kabel jenis ini mampu memiliki lebar pita yang lebih tinggi dan jarak yang lebih jauh daripada multimode dan sering digunakan untuk tulang punggung kampus. Kabel mode tunggal menggunakan laser sebagai metode penghasil cahaya dan lebih mahal daripada kabel multimode. Panjang kabel maksimum kabel mode tunggal adalah 60+ km (37+ mil).</li>
<li>Multimode </br>Kabel serat optik multimode memungkinkan beberapa mode cahaya masuk ke gerbang propa melalui serat. Kabel multimode sering digunakan untuk aplikasi grup kerja, menggunakan dioda pemancar cahaya (LED) sebagai perangkat penghasil cahaya. Panjang maksimum kabel multimode adalah 2 km (1,2 mil).</li></ul>
<p align="justify">Karakteristik media yang berbeda berdampak signifikan terhadap kecepatan transfer data. Meskipun kabel serat optik lebih mahal, kabel tersebut tidak rentan terhadap EMI dan memiliki kecepatan data yang lebih tinggi daripada jenis media jaringan lain yang dibahas di sini. Kabel serat optik juga lebih aman karena tidak memancarkan sinyal listrik yang bisa diterima oleh perangkat luar.</br></br>
<b>CATATAN</b></br>
Meskipun cahaya adalah gelombang elektromagnetik, cahaya dalam serat tidak dianggap nirkabel karena gelombang elektromagnetik dipandu dalam serat optik. Istilah nirkabel digunakan untuk gelombang elektromagnetik yang terpancar atau tidak terarah.</br></br>
Dalam beberapa kasus, mungkin tidak dapat menjalankan semua jenis kabel untuk komunikasi jaringan. Situasi ini mungkin terjadi di fasilitas sewaan atau di lokasi di mana Anda tidak memiliki kemampuan untuk memasang infrastruktur yang sesuai. Dalam kasus ini, mungkin berguna untuk menginstal jaringan nirkabel, seperti yang dibahas di bagian selanjutnya.</br></br>

Untuk LAN nirkabel, komponen kuncinya adalah hub nirkabel, atau titik akses, yang digunakan untuk distribusi sinyal. Untuk menerima sinyal dari titik akses, PC atau laptop perlu memasang kartu adaptor nirkabel, atau kartu antarmuka jaringan nirkabel (NIC). <p align="justify"><b>TCP</b></br>
TCP adalah protokol transport berorientasi koneksi yang mengirimkan data sebagai aliran byte tak terstruktur. Dengan menggunakan nomor urut dan pesan pengakuan, TCP dapat menyediakan node pengirim dengan informasi pengiriman tentang paket yang dikirimkan ke node tujuan. Jika data telah hilang dalam perjalanan dari sumber ke tujuan, TCP dapat mengirimkan kembali data tersebut hingga kondisi batas waktu tercapai atau hingga pengiriman berhasil dicapai. TCP juga dapat mengenali pesan duplikat dan akan membuangnya dengan semestinya. Jika komputer pengirim melakukan transmisi terlalu cepat untuk komputer penerima, TCP dapat menggunakan mekanisme kontrol aliran untuk memperlambat transfer data. TCP juga dapat mengkomunikasikan informasi pengiriman ke protokol lapisan atas dan aplikasi yang didukungnya. Semua karakteristik ini membuat TCP menjadi protokol transport yang andal dari ujung ke ujung. </br></br>
<b>IP </b>adalah protokol Layer 3 utama di suite Internet. Selain perutean internetwork, IP menyediakan pelaporan kesalahan dan fragmentasi serta pemasangan kembali unit informasi yang disebut datagram untuk transmisi melalui jaringan dengan ukuran unit data maksimum yang berbeda. IP mewakili inti dari rangkaian Protokol Internet.</br></br>
Catatan:  Istilah IP di bagian mengacu pada IPv4 kecuali dinyatakan lain secara eksplisit.</br></br>
Alamat IP unik secara global, nomor 32-bit ditetapkan oleh Pusat Informasi Jaringan. Alamat yang unik secara global mengizinkan jaringan IP di mana pun di dunia untuk berkomunikasi satu sama lain.</br></br>
Alamat IP dibagi menjadi dua bagian. Bagian pertama menunjukkan alamat jaringan sedangkan bagian kedua menunjukkan alamat host.</br></br>
Ruang alamat IP dibagi menjadi kelas jaringan yang berbeda. Jaringan Kelas A dimaksudkan terutama untuk digunakan dengan beberapa jaringan yang sangat besar, karena mereka hanya menyediakan 8 bit untuk bidang alamat jaringan. Jaringan Kelas B mengalokasikan 16 bit, dan jaringan Kelas C mengalokasikan 24 bit untuk bidang alamat jaringan. Jaringan kelas C hanya menyediakan 8 bit untuk bidang host, jadi jumlah host per jaringan dapat menjadi faktor pembatas. Dalam ketiga kasus tersebut, bit paling kiri menunjukkan kelas jaringan. Alamat IP ditulis dalam format desimal bertitik; misalnya, 34.0.0.1. Gambar dibawah ini menunjukkan format alamat untuk jaringan IP Kelas A, B, dan C.</p>
<p align="center"><img src="https://github.com/yenysyafitry/Cisco-CCNA/blob/main/13769-fig-2.png"> </p>
<p align="justify">Jaringan IP juga dapat dibagi menjadi beberapa unit yang lebih kecil yang disebut subnetwork atau "subnet". Subnet memberikan fleksibilitas ekstra untuk administrator jaringan. Misalnya, asumsikan bahwa jaringan telah diberi alamat Kelas A dan semua node di jaringan menggunakan alamat Kelas A. Selanjutnya asumsikan bahwa representasi desimal bertitik dari alamat jaringan ini adalah 34.0.0.0. (Semua angka nol di bidang host dari sebuah alamat menentukan seluruh jaringan.) Administrator dapat membagi jaringan menggunakan subnetting. Ini dilakukan dengan "meminjam" bit dari bagian host dari alamat dan menggunakannya sebagai bidang subnet, seperti yang digambarkan dibawah ini :</p>
<p align="center"><img src="https://github.com/yenysyafitry/Cisco-CCNA/blob/main/13769-fig-3.png"> </p>	
<p align="justify">Jika administrator jaringan telah memilih untuk menggunakan 8 bit subnetting, oktet kedua dari alamat IP Kelas A memberikan nomor subnet. Dalam contoh kami, alamat 34.1.0.0 mengacu pada jaringan 34, subnet 1; alamat 34.2.0.0 mengacu pada jaringan 34, subnet 2, dan seterusnya.
Jumlah bit yang dapat dipinjam untuk alamat subnet bervariasi. Untuk menentukan berapa banyak bit yang digunakan untuk merepresentasikan jaringan dan bagian subnet dari alamat, IP menyediakan subnet mask. Subnet mask menggunakan format dan teknik representasi yang sama dengan alamat IP. Subnet mask memiliki satu bit di semua bit kecuali yang menentukan kolom host. Misalnya, subnet mask yang menetapkan 8 bit subnetting untuk alamat Kelas A 34.0.0.0 adalah 255.255.0.0. Subnet mask yang menetapkan 16 bit subnetting untuk alamat Kelas A 34.0.0.0 adalah 255.255.255.0. Kedua subnet mask ini digambarkan pada Gambar dibawah berikut . Subnet mask dapat dilewatkan melalui jaringan sesuai permintaan sehingga node baru dapat mempelajari berapa banyak bit subnetting yang digunakan di jaringan mereka.</p>
<p align="center"><img src="https://github.com/yenysyafitry/Cisco-CCNA/blob/main/13769-fig-4.png"> </p>		
	<p align="justify">Secara tradisional, semua subnet dari nomor jaringan yang sama menggunakan subnet mask yang sama. Dengan kata lain, manajer jaringan akan memilih mask delapan-bit untuk semua subnet di jaringan. Strategi ini mudah dikelola untuk administrator jaringan dan protokol perutean. Namun, praktik ini membuang ruang alamat di beberapa jaringan. Beberapa subnet memiliki banyak host dan beberapa hanya memiliki sedikit, tetapi masing-masing menggunakan seluruh nomor subnet. Garis serial adalah contoh yang paling ekstrim, karena masing-masing hanya memiliki dua host yang dapat dihubungkan melalui subnet jalur serial.</br></br>
Seiring berkembangnya subnet IP, administrator telah mencari cara untuk menggunakan ruang alamat mereka dengan lebih efisien. Salah satu teknik yang dihasilkan disebut Variable Length Subnet Masks (VLSM). Dengan VLSM, administrator jaringan dapat menggunakan long mask pada jaringan dengan sedikit host dan short mask pada subnet dengan banyak host. Namun, teknik ini lebih kompleks daripada menjadikannya satu ukuran, dan alamat harus ditetapkan dengan hati-hati.</br></br>
Tentunya untuk menggunakan VLSM, seorang administrator jaringan harus menggunakan protokol routing yang mendukungnya. Router Cisco mendukung VLSM dengan Open Shortest Path First (OSPF), Integrated Intermediate System to Intermediate System (Integrated IS-IS), Enhanced Interior Gateway Routing Protocol (Enhanced IGRP), dan perutean statis.
Pada beberapa media, seperti IEEE 802 LAN, alamat IP secara dinamis ditemukan melalui penggunaan dua anggota lain dari suite protokol Internet: Address Resolution Protocol (ARP) dan Reverse Address Resolution Protocol (RARP). ARP menggunakan pesan siaran untuk menentukan alamat perangkat keras (lapisan MAC) yang sesuai dengan alamat lapisan jaringan tertentu. ARP cukup umum untuk memungkinkan penggunaan IP dengan hampir semua jenis mekanisme akses media yang mendasarinya. RARP menggunakan pesan siaran untuk menentukan alamat lapisan jaringan yang terkait dengan alamat perangkat keras tertentu. RARP sangat penting untuk node tanpa disk, yang alamat lapisan jaringannya biasanya tidak diketahui pada saat boot.</br></br>
<b>Perutean di Lingkungan IP</b></br>
Sebuah "internet" adalah sekelompok jaringan yang saling berhubungan. Internet, di sisi lain, adalah kumpulan jaringan yang memungkinkan komunikasi antara sebagian besar lembaga penelitian, universitas, dan banyak organisasi lain di seluruh dunia. Router di dalam Internet diatur secara hierarkis. Beberapa router digunakan untuk memindahkan informasi melalui satu grup jaringan tertentu di bawah otoritas dan kontrol administratif yang sama. (Entitas seperti itu disebut sistem otonom.) Router yang digunakan untuk pertukaran informasi dalam sistem otonom disebut router interior, dan mereka menggunakan berbagai protokol gateway interior (IGP) untuk mencapai tujuan ini. Router yang memindahkan informasi antara sistem otonom disebut router eksterior; mereka menggunakan Exterior Gateway Protocol (EGP) atau Border Gateway Protocol (BGP). menunjukkan arsitektur Internet.</p>

<p align="justify">Seperti yang telah kita lihat, perutean IP menentukan bahwa datagram IP melakukan perjalanan melalui internetwork satu lompatan router pada satu waktu. Seluruh rute tidak diketahui di awal perjalanan. Sebaliknya, di setiap perhentian, hop router berikutnya ditentukan dengan mencocokkan alamat tujuan dalam datagram dengan entri di tabel routing node saat ini. Keterlibatan setiap node dalam proses perutean hanya terdiri dari paket penerusan berdasarkan informasi internal. IP tidak menyediakan pelaporan kesalahan kembali ke sumber saat terjadi anomali perutean.</br></br>
ICMP melakukan sejumlah tugas dalam sebuah IP internetwork. Selain alasan utama pembuatannya (melaporkan kegagalan perutean kembali ke sumber), ICMP menyediakan metode untuk menguji jangkauan node di internet (pesan ICMP Echo dan Balas), metode untuk meningkatkan efisiensi perutean (ICMP Redirect message), metode untuk memberi tahu sumber bahwa datagram telah melebihi waktu yang dialokasikan untuk ada dalam internet (pesan ICMP Time Exceeded), dan pesan bermanfaat lainnya. Secara keseluruhan, ICMP merupakan bagian integral dari implementasi IP apa pun, terutama yang berjalan di router. </p>

<p align="justify"><b>Pahami Alamat IP</b>
Alamat IP adalah alamat yang digunakan untuk mengidentifikasi perangkat secara unik di jaringan IP. Alamatnya terdiri dari 32 bit biner, yang dapat dibagi menjadi bagian jaringan dan bagian host dengan bantuan subnet mask. 32 bit biner dipecah menjadi empat oktet (1 oktet = 8 bit). Setiap oktet diubah menjadi desimal dan dipisahkan oleh titik (titik). Untuk alasan ini, alamat IP dinyatakan dalam format desimal bertitik (misalnya, 172.16.81.100). Nilai di setiap oktet berkisar dari 0 hingga 255 desimal, atau 00000000 - 11111111 biner.</br></br>
Berikut adalah bagaimana oktet biner diubah menjadi desimal: Bit paling kanan, atau bit paling tidak signifikan, dari sebuah oktet memiliki nilai 2<sup>0</sup> . Bit di sebelah kiri yang memiliki nilai 2<sup>1</sup> . Ini berlanjut hingga bit paling kiri, atau bit paling signifikan, yang memiliki nilai 2<sup>7</sup> . Jadi jika semua bit biner adalah satu, padanan desimalnya adalah 255 seperti yang ditunjukkan di sini:</p>

```plantuml
    1 1 1 1 1 1 1 1
  128 64 32 16 8 4 2 1 (128 + 64 + 32 + 16 + 8 + 4 + 2 + 1 = 255)
```

<p align="justify">Dan contoh ini menunjukkan alamat IP yang direpresentasikan dalam biner dan desimal.</p>

```plantuml
 0 1 0 0 0 0 0 1
 0 64 0 0 0 0 0 1 (0 + 64 + 0 + 0 + 0 + 0 + 0 + 1 = 65)
```

<p align="justify">Berikut adalah contoh konversi oktet ketika tidak semua bit disetel ke 1.</p>

```plantuml
        10. 1. 23. 19 (desimal)
  00001010.00000001.00010111.00010011 (biner)
```
